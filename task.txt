ТЕХНИЧЕСКОЕ ЗАДАНИЕ
Планшетное приложение для остеопата / мануального терапевта
Версия 1.0  |  Май 2026

1. ОБЩАЯ ИНФОРМАЦИЯ

Название проекта	OsteoTab — CRM для специалиста по телу
Целевое устройство	Samsung Galaxy Tab S11 (Android, 14.6", 2960×1848 px)
Режим отображения	Альбомная и книжная ориентация (Landscape), поддержка S Pen / стилуса
Платформа	Progressive Web App (PWA) — устанавливается через браузер Chrome
Технологии	React + TypeScript + Tailwind CSS + localStorage (без серверной части)
Язык интерфейса	Русский
Голосовой ввод	Web Speech API (русский язык)
ИИ-интеграция	Anthropic Claude API — обработка заметок, AI-заключения
Хранение данных	localStorage (1-я фаза), Supabase (2-я фаза — опционально)


2. ЦЕЛЬ ПРОЕКТА
Создать удобное мобильное рабочее место специалиста (остеопата, мануального терапевта) для ведения карточек клиентов, документирования приёмов, отслеживания динамики лечения и аналитики. Приложение должно работать полностью офлайн на планшете Samsung Galaxy Tab S11 и устанавливаться как PWA без App Store.


3. СТРУКТУРА ПРИЛОЖЕНИЯ
Приложение состоит из 5 основных разделов, доступных через боковое меню слева:

№	Раздел	Описание
1.1	Список клиентов	Список всех клиентов с датой первого и последнего приёма. Поиск и фильтрация.
1.2	Карточка клиента	8 вкладок с полной информацией о клиенте и динамикой по приёмам
1.3	Аналитика	Графики посещений, прогресса, сравнение фото до/после
1.4	Календарь	Расписание приёмов, запись клиентов на следующий визит
1.5	Оплаты	Учёт стоимости, фактических оплат, задолженностей


4. ДЕТАЛЬНОЕ ОПИСАНИЕ МОДУЛЕЙ
4.1 Список клиентов (экран 1.1)
•	Таблица с колонками: ФИО, Возраст, Дата первого приёма, Дата последнего приёма, кнопка «Открыть»
•	Строка поиска по ФИО в верхней части экрана
•	Кнопка «+ Создать нового клиента» — открывает пустую карточку
•	Сортировка по дате последнего приёма (новые сверху)
•	Визуальный индикатор: если последний приём > 30 дней — жёлтый, > 60 дней — красный

4.2 Карточка клиента (экран 1.2) — 8 вкладок
Навигация по вкладкам располагается горизонтально в верхней части экрана. Все данные карточки привязаны к конкретному клиенту и хранятся в localStorage.

Вкладка 1 — Карточка клиента (1.2.1)
•	Поля: ФИО (текстовое), Возраст (числовое)
•	Загрузка фотографий с приёма — минимум 4 фото (все проекции для постурального анализа): фронтальная, сзади, слева, справа
•	Загрузка файлов анализов (PDF, фото)
•	Галерея загруженных фото с подписью даты
•	Кнопка голосового ввода для заполнения полей

Вкладка 2 — Анамнез (1.2.2)
•	Текстовые поля: Симптомы, Первые симптомы, Жалобы, Травмы, Шрамы, Препараты, Родовые травмы, Позы сна, Специалисты, Лечение, Результат лечения, Диагноз, Доп. информация, Результат который хотели бы
•	Три ползунка (1–10): Готовность тела, Готовность ума, Готовность сознания
•	Кнопка голосового ввода (Web Speech API, русский язык) — нажимает → говорит → текст автоматически вставляется в нужное поле

Вкладка 3 — Осмотр позвоночника (1.2.3)
•	Интерактивная SVG-диаграмма позвоночника (вид спереди и сбоку) с подписями позвонков: C1–C7 (шейный), T1–T12 (грудной), L1–L5 (поясничный), S1–S2 (крестцовый)
•	Нажатие / касание стилусом на позвонок — помечает его как «не в оси» (меняет цвет на оранжевый/красный)
•	Возможность нарисовать аннотацию стилусом поверх изображения (Canvas overlay)
•	Каждый приём сохраняется с датой — отображается временная шкала (Timeline) для отслеживания динамики
•	4 изображения скелета в разных проекциях как справочный материал

Вкладка 4 — Неврологические тесты (1.2.4)
•	Данные вводятся по каждому приёму отдельно (с датой)
•	Тесты: Поза Ромберга (Лев/Прав), Квадрат (текст), Пальце-носовая проба (Лев/Прав), Подвижность суставов, Устойчивость (Лев/Прав), Наклоны
•	Результат каждого теста: выпадающий список «норма / отклонение» или текстовое поле
•	Режим сравнения: 4 последних приёма отображаются в 4 колонках рядом для визуального сравнения динамики

Вкладка 5 — Центры тяжести (1.2.5)
•	Данные по каждому приёму с датой
•	4 паттерна: Паттерн дыхания, Паттерн правого шага, Паттерн левого шага, Паттерн вертикального положения
◦	Каждый паттерн: сетка 2×2, нажатие на ячейку переключает «+» / «–»
◦	Слева — эталонная схема для сравнения
•	Схема ступней (левая + правая нога) с нумерацией пальцев 1–5; нажатие отмечает положение мизинца
•	Поля веса: Вес кг, Вес 1/1, Вес затраты (Лев / Прав), Итого
•	Динамика: сравнение данных по нескольким приёмам

Вкладка 6 — Этажи, связи и проблема в регионах (1.2.6)
•	Две интерактивные диаграммы тела: спереди и сзади
•	Регионы спереди (Лев/Прав): Стопы, Колени, Тазобедренный сустав, Таз, Живот, Грудной отдел, Ключичный отдел, Шея, Голова, Локти, Кисти, Плечи
•	Регионы сзади (Лев/Прав): Пятки, Икроножные мышцы, Бёдра, Бедренные мышцы, Ягодичные мышцы, Поясница, Грудной отдел, Лопатки, Шейный отдел, Затылок
•	Нажатие переключает «+» (стабильно) / «–» (нестабильно)
•	Автоматическая логика: если с обеих сторон стоит «–» в одном регионе → выделяется красным как «первопричина»
•	Данные сохраняются по каждому приёму с датой — отслеживание динамики

Вкладка 7 — Мышечные цепи (1.2.7)
•	4 группы мышечных цепей с анатомическими изображениями
•	Для каждой цепи: сетка Лев / Прав по приёмам
•	Состояние каждой ячейки: Норма (зелёный) / Разрыв (красный) — переключается нажатием
•	Динамика: изменение состояния цепей от приёма к приёму визуализируется цветом

Вкладка 8 — Информация о приёме (1.2.8)
•	Блоки по датам приёмов (каждый новый приём — новый блок с датой)
•	Два текстовых поля в каждом блоке:
◦	«Что обсуждали / чем работали» — заметки по итогам приёма
◦	«Договорились на следующий раз» — план следующего визита
•	Кнопка голосового ввода на каждое поле (Web Speech API, русский язык)
•	Кнопка «AI-анализ» — отправляет текст в Claude API и получает структурированное заключение по приёму
•	Возможность редактирования записей прошлых приёмов


5. ДОПОЛНИТЕЛЬНЫЕ ЭКРАНЫ
5.1 Аналитика (1.3)
•	График посещений: количество приёмов по неделям/месяцам (линейный или столбчатый)
•	Динамика по параметрам: вес (Лев/Прав), стабильность регионов — сводный график
•	Сравнение фото До / После с подписями дат
•	Сводная таблица данных по всем приёмам клиента
•	Блок AI-заключения: Claude API анализирует всю историю клиента и формирует текстовый отчёт о прогрессе

5.2 Календарь (1.4)
•	Вид: месяц (по умолчанию) с возможностью переключения на неделю
•	Клик на дату — создание записи: выбор клиента из списка, время приёма, комментарий
•	Список предстоящих записей в боковой панели
•	Уведомление (badge) на иконке при наличии записей на сегодня

5.3 Оплаты (1.5)
•	Записи по каждому клиенту: Дата, Длительность (мин), Стоимость (₽), Фактически оплачено (₽), Разница / Долг
•	Итоговая строка: суммарный доход, общий долг
•	Данные оплат отражаются в разделе Аналитика


6. ТЕХНИЧЕСКИЕ ТРЕБОВАНИЯ
6.1 Общие
•	PWA (Progressive Web App): работает офлайн, устанавливается через Chrome → «Добавить на главный экран»
•	Адаптирован для Samsung Galaxy Tab S11: разрешение 2960×1848 px, альбомная книжная ориентация
•	Поддержка Samsung S Pen / стилуса: нажатие, рисование на Canvas, аннотации
•	Все касательные элементы (кнопки, ячейки) — минимум 44×44 px
•	Локальное хранение данных: localStorage (без необходимости интернета для работы)
•	Экспорт данных клиента в PDF (базовый — ФИО + данные по приёмам)

6.2 Голосовой ввод
•	Использование Web Speech API (встроено в Chrome)
•	Язык распознавания: русский (ru-RU)
•	Режим работы: нажал кнопку → говоришь → текст вставляется в активное поле
•	Индикатор записи (анимированный микрофон) во время распознавания

6.3 Интеграция с Claude AI
•	Используется Anthropic Claude API (claude-sonnet-4-20250514)
•	API-ключ хранится в настройках приложения (вводится пользователем один раз)
•	Вызов AI доступен в двух местах:
◦	Вкладка 1.2.8 — анализ одного приёма: краткое заключение по введённым заметкам
◦	Аналитика 1.3 — анализ всей истории клиента: сводный отчёт о динамике

6.4 Дизайн
•	Стиль: чистый медицинский, минималистичный
•	Цветовая схема: белый / светло-серый фон, тёмно-бирюзовый акцент (#1A6B72), тёмный текст (#1A1A2E)
•	Шрифт: системный (Arial / Roboto) — без подгрузки внешних шрифтов
•	Анимации: плавные переходы между вкладками (200–300 мс)
•	Тёмная тема: не требуется в 1-й версии


8. КРИТЕРИИ ПРИЁМКИ
•	Приложение устанавливается на Samsung Galaxy Tab S11 как PWA через Chrome
•	Работает полностью офлайн (все данные в localStorage)
•	Все 8 вкладок карточки клиента функционируют корректно
•	Голосовой ввод работает на русском языке
•	SVG-диаграмма позвоночника интерактивна — позвонки отмечаются нажатием
•	Динамика (временная шкала) отображается корректно в вкладках 1.2.3–1.2.7
•	AI-анализ приёма работает через Claude API
•	Нет критических ошибок при 30-минутном тестировании
•	Исходный код передаётся заказчику (GitHub / архив)
